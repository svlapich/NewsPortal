# AFN

Warning! Russian ahead.

Проект представляет из себя новостной портал, который загружает данные из [публичной страницы VK](https://vk.com/public179220995), используя VK API.

## Получение проекта
``` git clone https://github.com/MikalaiKazak/NewsPortal.git ```

## Запуск проекта
1. Установите nodejs - sudo apt install nodejs
2. Установите npm - sudo apt install npm
3. Установите gulp - sudo npm install --global gulp
4. В нужной директории введите run gulp -- gulp
5. Наслаждайтесь

# Использованные инструменты
* [NodeJS](https://nodejs.org/en/) - это среда выполнения JavaScript.
* [Gulp](https://gulpjs.com/) - это инструмент сборки веб-приложения, позволяющий автоматизировать повторяющиеся задачи, такие как сборка и минификация CSS и JS-файлов, запуск тестов, перезагрузка браузера и т.д. Тем самым Gulp ускоряет и оптимизирует процесс веб-разработки. 
* [VK API](https://vk.com/dev/manuals) - для работы со всеми методами API необходимо передавать в запросе access_token — специальный ключ доступа. Он представляет собой строку из латинских букв и цифр и может соответствовать отдельному пользователю, сообществу или приложению. В API ВКонтакте для получения ключа доступа используется открытый протокол [OAuth 2.0](https://oauth.net/2/). При этом пользователь не передает логин и пароль приложению, поэтому его аккаунт не может быть скомпрометирован. Используя токен, отправляется ajax запрос с методом wall.get.  wall.get возвращает список записей со стены пользователя или сообщества в формате JSON. Подробнее о методе, его возвращаемых параметрах и примерах применения можно почитать из официальной [документации](https://vk.com/dev/wall.get). Далее эти данные обрабатываются и содержимое записей на стене разносится по отдельным категориям на сайте.
* [Material Disign Lite](https://getmdl.io/started/index.html) - компоненты MDL создаются с использованием CSS, JavaScript и HTML. Он используется для создания веб-страниц и веб-приложений для создания более эстетичного, последовательного и функционального вида. Страницы, разработанные с использованием MDL, как минимум, придерживаются современных принципов веб-дизайна, типа переносимости браузера или независимости от устройства. Более общая информация об MDL есть на официальном [FAQ](https://getmdl.io/faq/index.html).
* [jQuery](https://api.jquery.com/) - библиотека JavaScript, фокусирующаяся на взаимодействии JavaScript и HTML. jQuery помогает легко получать доступ к любому элементу DOM, обращаться к атрибутам и содержимому элементов DOM, манипулировать ими. Библиотека также предоставляет удобный API для работы с AJAX, что и послужило его основной причиной включения в проект.
* [Saas](https://sass-lang.com/) - это расширение CSS, которое добавляет дополнительный функционал к стандартному языку. Он вводит поддержку переменных, вложенности и многого другого при полной поддержке CSS синтаксиса. Sass позволяет поддерживать большие таблицы стилей организованными и быстрыми.
* [000webhost](https://www.000webhost.com/) - Основное преимущество хостинга – его бесплатность. Скорость соединения также приемлема по современным стандартам.

# Структура проекта

### Общая структура
* github – это директория для работы с основными операциями github (например, pull request). Подробнее: https://help.github.com/en
*	app – это основная директория с файлами проекта.
*	.bowerrc – файл, указывающий путь установки дополнительных компонентов (/app/libs, в данном случае). Подробнее: https://bower.io/
*	.gitignore – содержит в себе список паттернов, файлов, директорий которые git не будет проверять. Подробнее: https://help.github.com/en/articles/ignoring-files
*	LICENSE – GNU General Public License v3.0.
*	README.md – базовая документация по проекту.
*	gulpfile.js – файл со списком задач. Подробнее: https://gulpjs.com/docs/en/getting started/quick start
*	package.json –  имеет базовую информацию о приложении (название, версия, зависимости и т. д.).

### Директория app
*	fonts – директория, где хранятся шрифты.
*	img – директория, где хранятся изображения.
*	js – директория, где хранится два js-файла:
    - script.min.js – это файл, который формирует gulp и он представляет собой конкатенацию всех js-файлов в проекте для ускорения загрузки страницы.
    - common.js – файл, где прописана основная логика приложения.
*	libs – директория, где хранятся библиотеки.
*	sass – директория для хранения файлов метаязыка на основе CSS (CSS with superpowers).
*	drawer.html, footer.html, header.html – шаблоны html.
*	index.html – основная html страница.
*	post.html – дополнительная html страница.
